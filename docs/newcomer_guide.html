项目概览

CodeMate AI 是一个智能代码分析助手，帮助开发者理解、分析和改进代码

Function Calling

使用 OpenAI 兼容的 API 格式，无需解析文本，LLM 直接返回结构化的 tool_calls，更可靠、更高效。

模块化工具

工具按类别组织（文件/搜索/Shell），易于扩展和维护。通过 ToolRegistry 实现工具的注册和调用。

Pydantic 验证

所有数据模型使用 Pydantic 进行验证，确保类型安全和数据一致性，提供更好的 IDE 支持。

三层日志架构

运行时日志 + Trace 轨迹日志 + Metrics 统计，完整的日志系统支持会话回放和成本追踪。

Token 统计

跟踪 API 使用量和预估成本，帮助控制成本和优化使用效率。

简洁 CLI

基于 Rich 的美观命令行界面，支持交互模式和单次查询，提供优秀的用户体验。

核心执行流程

1

接收用户输入

用户通过 CLI 输入问题，如"这个项目有哪些文件？"

2

调用 LLM（带工具列表）

Agent 将用户消息和可用工具列表发送给 GLM-4 

{
  "model": "glm-4-flash",
  "messages": [...],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "list_dir",
        "description": "列出目录内容",
        "parameters": {...}
      }
    }
  ]
}
3

LLM 返回工具调用

LLM 识别需要调用工具，返回结构化的 tool_calls 

{
  "tool_calls": [{
    "id": "call_abc123",
    "type": "function",
    "function": {
      "name": "list_dir",
      "arguments": '{"path": "."}'
    }
  }]
}
4

执行工具

Agent 通过 ToolRegistry 查找并执行工具，获取结果

5

将结果返回给 LLM

工具执行结果作为工具消息添加到对话历史

6

重复步骤 2-5

继续调用 LLM，直到 LLM 返回最终答案（tool_calls = None）

注意事项

使用 CodeMate AI 时需要注意的重要事项

安全注意事项

⚠️

危险工具需要确认

delete_file、write_file、run_shell 等危险工具需要用户确认后才执行

⚠️

工作目录限制

工具只能在指定的工作目录内操作，无法访问系统其他位置

⚠️

受保护文件

无法删除 .git、__pycache__ 等受保护的文件和目录

⚠️

API Key 保护

GLM_API_KEY 存储在 .env 文件中，不要提交到版本控制

性能与成本

💡

上下文压缩

当对话历史过长时，自动压缩上下文，节省 token 成本

💡

循环检测

检测并中断重复调用模式，避免浪费 token

💡

输出截断

过长的工具输出会被截断，避免发送过多 token 给 LLM

💡

最大轮数限制

默认最多 50 轮交互，防止无限循环消耗 token

常见问题

Q: 如何切换模型？

A: 使用 --model 参数，如 ./run.sh --model glm-4-plus

Q: 如何查看历史会话？

A: 查看 logs/traces/ 目录下的 trace-*.md 文件

Q: 工具执行失败怎么办？

A: 查看运行时日志和 Trace 日志，检查错误信息

Q: 如何添加自定义工具？

A: 继承 Tool 基类，实现 name、description、parameters 和 run 方法

快速开始

按照以下步骤快速启动 CodeMate AI

1

安装

# 克隆项目

git clone <repository-url>
cd codemate-agent

# 激活环境

conda activate codemate

# 安装依赖

pip install -r requirements.txt
2

配置

# 创建 .env 文件

cp .env.example .env

编辑 .env 文件：

GLM_API_KEY=your_api_key_here
GLM_MODEL=glm-4-flash
MAX_ROUNDS=50
TEMPERATURE=0.7

LOG_LEVEL=INFO
TRACE_ENABLED=true
TRACE_DIR=logs/traces
METRICS_ENABLED=true
METRICS_DIR=logs/sessions
3

运行

# 交互模式

./run.sh

# 单次查询

./run.sh "分析 examples/"

# 指定模型

./run.sh --model glm-4-plus
