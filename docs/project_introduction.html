项目概览

CodeMate AI 是一个帮助开发者快速理解和分析代码的智能助手

🎯

项目目标

帮助开发者快速理解陌生项目的代码结构、搜索和分析代码内容、获取重构建议和最佳实践、自动化代码审查流程。

💡

核心理念

采用原生 Function Calling 范式，通过结构化的工具调用实现更可靠的代码分析，告别文本解析的不确定性。

✅ 原生 Function Calling
✅ 模块化工具系统
✅ Pydantic 数据验证
✅ 简洁的 CLI 界面
✅ 三层日志架构
✅ Token 统计

🤖 Agent 核心

基于 Function Calling 的智能代理，负责协调工具调用和对话管理

🔧 工具系统

模块化的工具注册和执行机制，支持文件操作、代码搜索、命令执行等

📊 日志系统

三层日志架构：运行时日志、Trace 轨迹、Metrics 统计

💾 持久化

会话存储、记忆管理、会话索引，支持长期记忆和会话回放

🧠 任务规划

自动检测复杂任务，生成执行计划，跟踪任务进度

⚡ Skill 系统

预定义的专业任务流程，如代码审查、UI/UX 设计等

工作流程

从用户输入到最终答案的完整流程

1

用户输入

用户通过 CLI 输入问题或任务

./run.sh "分析这个项目"
→
2

构建请求

Agent 构建包含工具列表的 LLM 请求

messages + tools → LLM API
→
3

LLM 推理

GLM-4 模型分析请求，决定是否调用工具

tool_calls: [list_dir, path: "."]
→
4

工具执行

执行工具并获取结果

result: ["codemate_agent/", "tests/", ...]
→
5

结果返回

将工具结果返回给 LLM 继续推理

tool message → LLM
→
6

最终答案

LLM 生成最终答案并返回给用户

"这个项目包含以下目录..."
💡 关键特性：
  • Function Calling：LLM 直接返回结构化的 tool_calls，无需文本解析
  • 自动循环：Agent 自动处理多轮工具调用，直到获取最终答案
  • 上下文管理：维护完整的对话历史，支持多轮交互

注意事项

使用 CodeMate AI 时需要了解的重要信息

🔒 安全注意事项

⚠️

Shell 命令注入风险

Shell 工具存在命令注入风险，建议使用命令白名单而非黑名单过滤

⚠️

路径遍历漏洞

文件操作工具可能访问项目外文件，需要添加路径验证

⚠️

API 密钥安全

请妥善保管 GLM_API_KEY，不要提交到版本控制系统

⚙️ 配置说明

ℹ️

环境变量配置

创建 .env 文件并配置 GLM_API_KEY、GLM_MODEL 等参数

ℹ️

模型选择

glm-4-flash 速度快成本低，glm-4-plus 能力更强成本更高

ℹ️

日志配置

支持运行时日志、Trace 轨迹和 Metrics 统计三种日志类型

📚 使用建议

✅

从简单任务开始

先尝试简单的代码分析任务，熟悉工具的使用方式

✅

查看 Trace 日志

通过 Trace 日志了解 Agent 的决策过程和工具调用情况

✅

利用 Skill 系统

使用预定义的 Skill（如 code-review）快速完成专业任务

🐛 已知问题

❌

API 调用无重试

网络不稳定时可能导致任务失败，建议手动重试

❌

测试覆盖率不足

核心模块测试覆盖率约 17%，建议谨慎使用

❌

类型提示不一致

部分代码使用 list[str]，部分使用 List[str]，需要统一

cp .env.example .env && 编辑 .env
3
