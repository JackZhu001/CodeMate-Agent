# CodeMate 记忆系统与上下文工程（详细说明）

本文档描述 CodeMate 当前的记忆分层、压缩机制、落盘策略，以及下一步可演进方向。

## 1. 设计目标

1. 在长对话里尽量不丢任务主线  
2. 在上下文接近上限时，平稳降载而非“硬截断”  
3. 对历史信息做到可回溯（summary + transcript）  
4. 让用户可主动控制（`/compact`）

---

## 2. 三层记忆模型

## 2.1 短时记忆（Short-term）

**定义**：当前会话内、直接送入 LLM 的消息历史。  
**实现载体**：`self.messages`（`Message` 列表）。

短时记忆承担两类信息：
- 对话轮次（user / assistant）
- 工具调用反馈（tool）

---

## 2.2 工作记忆（Working Memory）

**定义**：任务执行状态（例如 todo 进度、已完成项、阻塞项）。  
**实现方式**：在压缩过程中注入当前 TODO 状态，降低任务状态丢失概率。

建议实践：
- 将里程碑与未完成项维持为结构化文本
- 在每次重大状态变化后更新
- 让“状态事实”与“自然语言对话”分离

---

## 2.3 长时记忆（Long-term）

**定义**：跨会话持久信息。  
**实现方式**：
- 会话数据持久化
- 记忆文件可通过 `/memory` 查看
- 自动压缩时完整 transcript 落盘，供后续回读

---

## 3. 压缩系统（Context Compressor）

关键文件：`codemate_agent/context/compressor.py`

## 3.1 Layer 1: Micro Compact（每轮微压缩）

目标：控制工具输出膨胀，优先保留最近轮次的可操作信息。

行为：
1. 识别对话轮次  
2. 只处理 `role="tool"` 消息  
3. 默认保留最近 3 轮工具输出原文  
4. 更早工具输出替换为占位提示（提示可回读）  
5. 白名单工具（默认如 `todo_write`）不压缩

价值：
- 显著降低旧工具输出占用
- 减少“上下文被日志刷爆”的情况

---

## 3.2 Layer 2: Auto Compact（自动压缩）

触发策略：
- 默认阈值：`context_window * 0.75`
- 默认窗口：200k（可配置）

执行策略：
1. 先把完整会话写入 transcript 文件  
2. 分离 `system` 与非 system 消息  
3. 保留最近 3 轮完整消息  
4. 将更早历史生成语义摘要  
5. 将摘要作为系统消息写回上下文，并附带“可回读 transcript”提示

价值：
- 历史“可浓缩但可追溯”
- 避免压缩后完全失去近期操作细节

---

## 3.3 Layer 3: Manual Compact（手动压缩）

CLI 支持 `/compact`，用户可在任意时刻手动触发压缩。

适用场景：
- 你知道下一步要进入大文件改造
- 准备开启新子任务，希望先“清场”

---

## 4. 配置项建议

常见参数：
- `CONTEXT_WINDOW`：上下文窗口
- `COMPRESSION_THRESHOLD`：压缩阈值比例（默认 0.75）
- `MICRO_COMPACT_KEEP`：微压缩保留轮数（默认 3）
- `MICRO_COMPACT_TOOL_WHITELIST`：微压缩白名单工具（逗号分隔）
- `TOKEN_THRESHOLD`：若 >0，优先使用固定 token 阈值

建议：
- 多工具密集场景：提高 `MICRO_COMPACT_KEEP` 到 4~6
- 大仓库深度任务：维持 `COMPRESSION_THRESHOLD` 在 0.70~0.80

---

## 5. 参考业界实践（简述）

结合 Claude Code / Codex / Aider 的公开资料，常见共识是：

1. **会话上下文是短时的**，需要外部文件承接长期规则/偏好  
2. **文档化记忆优先**（例如项目级规则文件 + 自动记忆索引）  
3. **按需读取细节**，避免把所有历史细节一直塞在 prompt 里

CodeMate 当前实现已经覆盖：
- 会话压缩
- transcript 可回读
- 手动压缩入口

下一步可继续演进：
- 引入结构化摘要索引（主题/决策/问题/结果）
- 加上关键词或向量检索层，实现“先召回摘要，再按需回读原文”

---

## 6. Roadmap（建议）

1. **工作记忆结构化固化**  
   将 todo / 阻塞 / 下一步分离为固定状态块，不参与摘要压缩。

2. **长时记忆索引化**  
   会话结束后生成结构化摘要条目（可检索）。

3. **用户画像与偏好文档**  
   引入 `USER_PROFILE.md` / `PREFERENCES.md`，在每轮构造上下文时注入。

4. **检索增强上下文拼装**  
   对历史摘要做召回，命中后再回读 transcript 片段，减少无关 token 开销。
